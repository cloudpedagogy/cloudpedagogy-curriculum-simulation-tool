# PROJECT_SPEC: cloudpedagogy-curriculum-simulation-tool

## 1. Repo Name
`cloudpedagogy-curriculum-simulation-tool`

## 2. One-Sentence Purpose
A dynamic simulator for modeling student workload, skill trajectories, and assessment pressure curves throughout an academic term.

## 3. Problem the App Solves
Static curriculum maps fail to account for the "Time" dimension; this tool visualizes how students actually experience the curriculum week-to-week, uncovering "assessment bunching" and inconsistent workload patterns.

## 4. Primary User / Audience
Programme designers, student experience leads, and educational researchers.

## 5. Core Role in the CloudPedagogy Ecosystem
The "Dynamic Simulator"; it builds upon the structural data from the Mapping Engine to project time-series and growth-series data.

## 6. Main Entities / Data Structures
- **Module**: Educational units with associated learning outcomes and skills.
- **Skill**: Capability units with a numeric proficiency value (0-100).
- **Assessment**: Evaluation tasks assigned to specific delivery weeks.
- **WorkloadProfile**: Weekly hours assigned to specific modules over the term.
- **SimulationDataset**: The unified dataset containing modules, assessments, skills, and workload distributions.

## 7. Main User Workflows
1. **Scenario Loading**: Import a baseline curriculum dataset.
2. **Timeline Management**: Assign assessment dates (Weeks 1-12) and define weekly study hours.
3. **Simulation Review**: Analyze the "Pressure" and "Skill" curves generated by the assignments.
4. **Scenario Overriding**: Create "What-if" variants of the curriculum to see the impact of design changes on student workload.

## 8. Current Features
- Weekly workload distribution (0-15+ hours).
- Numeric skill level projection (0-100).
- Timeline-based assessment positioning.
- Real-time visualization of workload pressure.
- Dual-state management (Base Dataset vs. Scenario Override).

## 9. Stubbed / Partial / Incomplete Features
- Scenario override logic is implemented but lists secondary data states as a work-in-progress for complex multi-variant testing.

## 10. Import / Export and Storage Model
- **Storage**: Browser `localStorage` (`curriculum_sim_current_dataset` and `curriculum_sim_scenario_state`).
- **Import/Export**: JSON-based `SimulationDataset` serialization.

## 11. Relationship to Other CloudPedagogy Apps
Consumer of structural curriculum node/edge data produced by the `curriculum-alignment-mapping-engine`.

## 12. Potential Overlap or Duplication Risks
Similar entity names to the mapping engine; distinguished by its addition of "Time" (Weekly distribution) and "Numeric Proficiency."

## 13. Distinctive Value of This App
The only tool in the ecosystem that treats the curriculum as a dynamic, time-bound experience rather than a static graph.

## 14. Recommended Future Enhancements
(Inferred) Student-persona modeling (e.g. "Full-time" vs "Part-time" vs "Commuter" workload profiles); automated detection of "Assessment Crisis" points.

## 15. Anything Unclear or Inferred from Repo Contents
The exact growth function for Skill Levels (linear vs. log) is inferred to be a simplified qualitative model in the current simulation engine.
